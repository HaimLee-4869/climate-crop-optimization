# 기후변화 하 미국 옥수수 수확량 예측 & 작물 배분 최적화 (v3)

> **프로젝트 제목:** 온난화 시나리오 하에서 예측 기반 작물 배분 최적화
> **데이터:** 미국 카운티 단위 농업·기후 패널 (1981–2015)
> **구성:** 머신러닝 2개 모델 + 최적화 (조합 최적화 / 메타휴리스틱)
>
> ※ v2 대비 변경점: (1) 교수님 피드백 반영해 ML 모델을 "수확량 예측 + 기후 예측" **두 개**로 확장, (2) 최적화를 순수 LP에서 **조합 최적화 + 메타휴리스틱(GA/SA)** 으로 전환해 강의 내용 반영, (3) 온난화 시나리오를 임의 +2°C 이동이 아니라 **기후 예측 모델의 출력**으로 대체.

---

## 0. 한눈에 보는 요약

이 프로젝트는 **세 단계**로 구성된다.

1. **수확량 예측 (ML 모델 B)** — 온도 노출 분포·강수·토양으로 카운티별 옥수수 수확량을 예측하고 여러 모델을 비교한다. 이 과정에서 알려진 "약 29°C 고온 피해 임계점"이 데이터에서 재현되는지 확인한다.
2. **기후 예측 (ML 모델 A)** — 연도·지역 추세를 학습해 미래의 온도 노출 분포·강수를 예측(=시나리오 생성)한다. 이것이 온난화 상황의 입력이 된다.
3. **작물 배분 최적화 (Optimization)** — 예측 수확량을 계수로, 제한된 농지에 어떤 작물을 배치할지를 **조합 최적화**로 푼다. 각 카운티가 주력 작물 하나를 고르는 단작(單作) 문제로 두어, 정확한 솔버(MILP)와 **메타휴리스틱(GA·SA)** 의 성능을 비교한다. 이어서 기후 예측 모델이 만든 온난화 시나리오로 재예측·재최적화하여 최적 작물 지도가 어떻게 바뀌는지(재배 벨트 북상) 정량화한다.

**핵심 메시지:** *"날씨로 수확량을 예측하고, 알려진 고온 임계점을 데이터에서 되찾으며, 기온이 오를 때 밭을 최소한으로만 바꿔도 손실을 얼마나 줄일 수 있는지 보여준다."*

## 1. 배경 (Background)

### 1.1 왜 이 주제인가

퍼듀는 농업(특히 옥수수)에서 강한 대학이고, 주변이 실제 옥수수 벨트다. 지역 특성을 살린 주제다. 스마트팜·기후적응 농업은 앞으로도 유망한 분야이며, 사이드 프로젝트로 흔히 나오는 뻔한 주제를 피하면서도 "퍼듀에서 이런 걸 했다"고 했을 때 설득력 있는 주제를 골랐다.

### 1.2 미국의 위치: 부족이 아니라 효율의 문제

미국은 식량 안보가 세계에서 가장 튼튼한 축이며, 옥수수·대두·밀을 국내 소비보다 훨씬 많이 생산해 대량 수출한다. 그럼에도 이 주제가 의미 있는 이유:

- 미국은 세계 곡물 시장의 최대 공급자라, 미국 생산량이 흔들리면 수입국들의 식량 가격·안보가 출렁인다. 미국 수확량은 글로벌 공급망 문제다.
- 이 프로젝트는 "부족을 막자"가 아니라 **"온난화 상황에서 같은 땅에 무엇을 어디에 심는 게 최적인가"라는 효율·적응 문제**를 다룬다. 최적화 문제로 딱 맞는다.

### 1.3 핵심 통찰: 평균기온이 아니라 "온도 노출 분포"

수확량-기온 관계를 다룰 때 흔한 실수는 생육기간 **평균기온** 하나로 요약하는 것이다. 평균이 같아도 극한 고온 노출은 완전히 다를 수 있다. 평균 24°C인 두 시즌 중 하나는 29°C를 거의 안 넘겼고 다른 하나는 며칠 35°C까지 치솟았을 수 있는데, 수확량을 결정적으로 망치는 것은 후자의 **극한열 노출**이다. 평균만 보면 이게 사라진다.

이 데이터는 각 카운티·연도의 생육기간 동안 작물이 **각 온도 구간에 얼마나 노출됐는지**를 분포 전체로 기록한다(`gddm60`…`gdd0`…`gddp60`, 총 121구간). 이는 Schlenker & Roberts (2009, PNAS)의 방법론으로, 옥수수는 약 29°C까지는 기온이 오를수록 수확량이 늘지만 그 임계점을 넘으면 매우 가파르게 감소하는 **비선형·비대칭 반응**을 밝혔다.

### 1.4 방법론 프레임: 예측 후 최적화 + 전환비용

예측 모델만으로는 "그래서 뭘 해야 하나"에 답하지 못한다. 예측 수확량을 의사결정(작물·토지 배분)에 연결한다(predict-then-optimize). 다만 순진한 최적화는 "이론적 최대 생산"만 좇아 현재 밭을 다 갈아엎으라는 비현실적 해를 낸다. 그래서 **현재 배분에서 벗어난 정도에 벌점을 주는 전환비용 항**을 넣어, "생산은 늘리되 지금과 너무 멀어지지 않는" 해를 구한다.

## 2. 강의 내용과의 연결 (왜 이 최적화 설계인가)

이 프로젝트는 최적화 강의에서 배운 두 축을 모두 활용하도록 설계했다.

- **정확한 방법 (LP / MILP):** 작물 배분을 선형계획으로 정식화. 절댓값 전환비용의 **선형화**, λ 스윕을 통한 **민감도 분석** 등 Lecture 1~3 내용을 직접 사용한다.
- **메타휴리스틱 (GA / SA):** 단작 제약을 넣으면 배분이 **조합 최적화(binary 배정)** 가 되어, 카운티 수가 커질수록 정확한 솔버가 느려진다. 이 지점이 Lecture 3~4의 "MILP의 한계 → 메타휴리스틱의 필요성" 서사와 정확히 맞물린다. GA(Lecture 7)와 SA(Lecture 6)로 '충분히 좋은' 작물 지도를 빠르게 찾고, **정확한 해 vs 근사 해**를 비교한다(Lecture 3의 "Traditional Solver vs Heuristic" 비교표를 실제 숫자로 채운다).

**진행 전략 (A → C 확장):** 위험을 낮추기 위해 단계적으로 확장한다.

1. **1단계 (필수):** GA로 단작 배분을 완성 → 이 시점에 이미 제출 가능한 완성품 확보.
2. **2단계 (여유 시):** 같은 문제를 SA로도 풀어 GA와 비교.
3. **3단계 (추가 여유 시):** 필요 시 Tabu Search까지 추가하고, 다회 반복 실행으로 통계적 공정 비교(SA 하이퍼파라미터 튜닝 자료의 실험설계 적용: 설정별 10~15회 반복, 평균±표준편차).

어느 단계에서 시간이 끊겨도 그 지점까지가 완결된 결과물이 되도록 한다.

## 3. 연구 질문과 가설

**연구 질문**

1. 관측된 온도 노출·강수·토양만으로 카운티 옥수수 수확량을 얼마나 잘 예측하는가?
2. 알려진 고온 임계점(약 29°C)이 데이터에서 재현되는가?
3. 지역·연도 추세로 미래 기후(온도 노출 분포)를 얼마나 그럴듯하게 예측하는가?
4. 예측 기반 최적 배분은 현재 배분보다 총생산을 얼마나 높이는가?
5. 정확한 솔버(MILP)와 메타휴리스틱(GA/SA)은 해의 품질·계산 시간에서 어떻게 다른가?
6. 전환비용을 넣으면 배분 변화량과 생산 개선 사이에 어떤 트레이드오프가 나타나는가?
7. 온난화 시나리오에서 최적 작물 지도는 어떻게 바뀌며, 최소한의 변화로 손실을 얼마나 회복하는가?

**가설**

- **H1.** 온도 노출·강수·토양은 옥수수 수확량을 유의미하게 예측한다(교차검증 R²이 상당 수준).
- **H2.** 약 29°C 부근에서 수확량이 꺾이는 비선형·비대칭 반응이 관측된다.
- **H3.** 온도 구간이 서로 강하게 상관되어, 정규화 모델(Ridge/Lasso)·트리 모델(RF/GBM)이 단순 OLS보다 안정적이고 정확하다.
- **H4.** 기후 예측 모델은 장기 상승 추세와 지역 차이를 재현하되, 미래로 갈수록 불확실성이 커진다(단일 예측이 아니라 시나리오로 다뤄야 한다).
- **H5.** 메타휴리스틱(GA/SA)은 정확한 솔버 대비 훨씬 빠르게 최적해에 근접한 해를 찾는다. 문제 규모가 커질수록 이 격차가 벌어진다.
- **H6.** 전환비용 λ를 키우면 배분 변화량은 줄지만 생산 개선폭도 줄어드는 **우하향 트레이드오프 곡선**이 나오고, 중간에 "적은 변화로 큰 개선"의 스윗스팟이 존재한다.
- **H7.** 온난화 시 무조정이면 총생산이 감소하지만, 최적 재배치로 손실의 상당 부분을 회복할 수 있다.
- **H8.** 온난화 시 최적 작물 지도가 북쪽으로 이동한다(재배 벨트의 북상).

## 4. 데이터

공통 키는 `stco`(5자리 FIPS = 주 2자리 + 카운티 3자리), 시계열 파일은 `year`(1981–2015)를 추가 키로 갖는다.

| 파일 | 내용 | 역할 |
| --- | --- | --- |
| `gddMarAug.csv`, `gddAprOct.csv` | 온도 노출 분포 (3–8월 / 4–10월), 121구간 | **핵심 입력.** "29°C 이하 유익 노출 / 초과 유해 노출"로 압축하거나 원자료+정규화로 사용 |
| `pptMarAug.csv`, `pptAprOct.csv` | 생육기간 강수량 `ppt`(mm) | 입력 변수(선형 + 이차항) |
| `soil1992/2001/2006/2011.csv` | 토양 특성(보수력, 모래·미사·점토, 유기물, 침식, pH, 경사 등) | 입력 변수(거의 시불변, 가까운 스냅샷 병합) |
| `yielddata.csv` | 옥수수·대두·면화·밀 수확량 | **타깃(예측 대상).** 또한 "현재 어떤 작물을 심나"의 근거(값 있으면 재배, NA면 미재배) |
| `gridInfo.csv` | 카운티 내 농지 규모 `numAg…` | 최적화에서 카운티별 **가용 농지량 A_c** 대용치, 현재 배분 근사에도 사용 |
| (외부) 기상 데이터 | 장기 기온·강수 시계열 등 | **기후 예측 모델(모델 A)** 학습 보강용. 교수님 조언대로 외부에서 확보 |

**분석 시 유의점**

- **결측치:** `yielddata`에 NA가 많다(서부 건조 카운티는 미재배 — 예: 와이오밍 56xxx는 옥수수 대부분 NA). 옥수수·대두→중서부, 면화→남부, 밀→대평원으로 지역을 좁힌다. **옥수수부터 시작**(데이터 많고 임계점 스토리 뚜렷).
- **생육창:** 옥수수는 보통 3–8월(`MarAug`). 하나를 골라 일관되게 사용.
- **단위:** 수확량 단위는 옥수수·밀·대두 bushels/acre, 면화 lbs/acre로 추정. 최종 해석 전 출처 확인.
- **"현재 배분" 근사:** 작물별 실제 재배 면적이 없으므로, 최근 몇 년 각 카운티의 실제 재배 작물(비-NA)을 파악하고 `gridInfo`의 농지 규모로 현재 배분 x̄를 근사한다(§9 한계에 명시).

## 5. 분석 파이프라인

### Phase 0 — 데이터 준비 및 EDA
- 모든 파일을 `stco`(+`year`)로 병합해 카운티×연도 패널 생성. 대상 작물을 옥수수로 확정.
- 볼 것: 수확량의 지역·연도 추세(기술 발전에 따른 장기 상승), 결측 패턴 지도, 온도 노출 분포의 형태, 수확량 vs 유해 고온 노출 산점도, 온도 구간 간 상관행렬(다중공선성 → 정규화 근거).

### Phase 1 — 피처 엔지니어링
두 전략을 모두 만들어 비교한다.
- **(A) 도메인 압축(Schlenker–Roberts):** `GDD_below`(임계점 이하 유익 노출 합), `GDD_above`(임계점 초과 유해 노출 합), `ppt`, `ppt²`, 토양 변수. 임계점을 28~31°C로 바꿔가며 최적점 탐색 → "임계점이 데이터에서 드러나는가"에 직접 답.
- **(B) 원자료 + 정규화:** 온도 구간 전체를 특징으로 넣고 Ridge/Lasso로 공선성 제어.

### Phase 2 — 수확량 예측 모델링 (ML 모델 B)
동일한 교차검증 프로토콜로 여러 모델 비교.

| 모델 | 학습 = 어떤 최적화 문제 | 역할 |
| --- | --- | --- |
| OLS 선형회귀 | 최소제곱(잔차 제곱합 최소화) | 베이스라인 |
| Ridge | 최소제곱 + L2 벌점 → QP | 공선성 제어 |
| Lasso | 최소제곱 + L1 벌점 → 볼록최적화 | 변수 선택(어떤 온도 구간이 중요한지) |
| Random Forest / Gradient Boosting | 탐욕적 분할 최적화 | 비선형·상호작용 자동 포착, 성능 상한 |

- **검증:** 시간 분할(예: 1981–2010 학습 / 2011–2015 평가) 또는 공간 교차검증. 지표는 RMSE, MAE, R².
- **해석:** 추정된 온도 반응곡선을 그려 29°C 부근 꺾임 확인(핵심 그림).
- **연결 포인트:** "모델 학습 자체가 최적화 문제"임을 명시 → ML과 최적화가 한 뿌리.

### Phase 3 — 기후 예측 모델링 (ML 모델 A)
- **목표:** 연도·카운티를 입력으로 미래의 온도 노출 분포·강수를 예측. 시계열 성격이 강하므로 추세 + 지역 효과를 잡는 모델 사용.
- **포지셔닝:** 1981–2015로 미래를 예측하는 것은 본질적으로 외삽이므로, "정밀한 예언"이 아니라 **시나리오 생성기**로 다룬다. 낙관·중립·비관 등 여러 시나리오를 뽑는다.
- **출력:** 각 시나리오별 미래 온도 노출 분포 → Phase 2 모델에 입력해 미래 수확량 예측.

### Phase 4 — 기본 최적화 (정확한 방법: LP / MILP)
- **연속 LP:** 예측 수확량 y_hat을 계수로 카운티별 농지 한도 안에서 총생산 최대화(정식화 §6). 이론적 최적(비교 기준).
- **단작 MILP:** "각 카운티는 작물 하나만" 제약(binary)을 추가 → 조합 최적화. 정확한 솔버로 풀되, 규모가 커질 때의 계산 시간을 기록(메타휴리스틱 필요성의 근거).

### Phase 5 — 메타휴리스틱 최적화 (핵심: A → C)
- **GA (1단계, 필수):** 염색체 = 카운티별 작물 선택 벡터, 적합도 = 총생산 − 전환비용, 연산자 = 선택·교차·돌연변이(일부 카운티 작물 변경). 은행 대출 예제(Lecture 10) 구조와 유사.
- **SA (2단계):** 동일 문제를 이웃 정의(일부 카운티 작물 스왑) + 냉각 스케줄로 풀어 GA와 비교.
- **(선택) TS (3단계):** tabu list로 재방문 방지. 세 방법을 다회 반복 실행하여 평균±표준편차로 공정 비교.
- **비교 지표:** 해의 품질(총생산), 계산 시간, 안정성(반복 간 분산), 수렴 속도.

### Phase 6 — 전환비용 트레이드오프
- 현재 배분 x̄에서 벗어난 정도에 벌점(λ). λ를 0부터 크게까지 스윕 → 트레이드오프 곡선(x축: 배분 변화량, y축: 총생산 개선).
- 곡선의 무릎(knee)에서 "작은 변화로 큰 개선"의 스윗스팟 식별. 실제 달러 비용 없이도 λ 스윕만으로 의사결정 정보를 제공.

### Phase 7 — 기후 시나리오 및 적응
- 모델 A가 만든 온난화 시나리오로 특징 재계산 → 수확량 재예측 → (전환비용 포함) 재최적화.
- 비교: (1) 무조정 시 손실 vs (2) 최소 변화 재배치로 회복한 양 → 적응의 가치.

### Phase 8 — 결과 해석 및 시각화
- 온도 반응곡선(임계점), 모델 성능 비교표, 예측 vs 실측 산점도
- 정확한 해 vs 메타휴리스틱 해 비교표(품질·시간·안정성)
- 전환비용 트레이드오프 곡선
- 카운티 코로플레스 지도: 기준 기후 vs 온난화 시나리오의 최적 작물 지도 → 재배 벨트 북상 시각화(FIPS로 카운티 경계 결합)

## 6. 최적화 정식화

**기호 정의**

```
c : 카운티        k : 작물(옥수수, 대두, ...)
x[c,k]     : (결정변수) 카운티 c에서 작물 k에 배정하는 농지 면적,  x[c,k] ≥ 0
z[c,k]     : (결정변수, 단작용) 카운티 c가 작물 k를 주력으로 고르면 1, 아니면 0
y_hat[c,k] : ML 모델이 예측한 c에서 k의 단위면적당 수확량
A_c        : 카운티 c의 가용 농지량 (gridInfo의 numAg 대용)
x_bar[c,k] : 현재 배분 = 지금 c에서 k에 심고 있는 면적 (데이터에서 근사)
lambda     : 전환비용 벌점의 세기 (실험 변수, ≥ 0)
```

### (1) 기본 버전 — 연속 LP (전환비용 없음)

```
maximize    Σ_c Σ_k  y_hat[c,k] · x[c,k]

subject to  Σ_k x[c,k] ≤ A_c          for all c     (카운티별 농지 한도)
            x[c,k] ≥ 0                 for all c,k   (음이 아님)
```

### (2) 단작 버전 — 조합 최적화 (MILP, 메타휴리스틱 대상)

각 카운티는 작물 하나만 선택하며, 그 작물에 카운티 농지 전체를 배정한다.

```
maximize    Σ_c Σ_k  y_hat[c,k] · A_c · z[c,k]

subject to  Σ_k z[c,k] = 1            for all c     (카운티당 작물 하나)
            z[c,k] ∈ {0, 1}           for all c,k   (binary 선택)
```

- 카운티 수가 커지면 z의 조합이 폭발 → 정확한 MILP 솔버 느려짐 → **GA/SA로 근사**.
- GA 염색체 = 각 카운티의 작물 인덱스 벡터(예: `[옥수수, 대두, 옥수수, 밀, ...]`).

### (3) 전환비용 버전 — 지금과 너무 달라지지 않게 (연속 LP + 선형화)

목적함수에 "현재 배분에서 바뀐 양"에 벌점을 뺀다. 절댓값은 보조변수 d로 선형화한다.

```
보조변수 d[c,k] ≥ 0 도입, 절댓값을 두 부등식으로 대체:

maximize    Σ_c Σ_k y_hat[c,k]·x[c,k]  −  lambda · Σ_c Σ_k d[c,k]

subject to  d[c,k] ≥ x[c,k] − x_bar[c,k]      for all c,k
            d[c,k] ≥ x_bar[c,k] − x[c,k]      for all c,k
            Σ_k x[c,k] ≤ A_c                   for all c
            x[c,k] ≥ 0,   d[c,k] ≥ 0           for all c,k
```

- `lambda = 0`: 전환비용 무시 → 이론적 최적 배분
- `lambda` 큼: 거의 안 바꾸는 보수적 배분
- `lambda` 스윕: 트레이드오프 곡선 → 분석의 핵심 그림

> 단작(2)에 전환비용을 넣으려면 "현재 작물과 다르게 고른 카운티 수"에 벌점을 주면 된다(메타휴리스틱 적합도 함수에 자연스럽게 추가).

### (선택) 현실성·다양성 제약

```
z[c,k] = 0                       (특정 지역에서 재배 불가한 작물)
Σ_c A_c·z[c,k] ≥ D_k             (작물 k 최소 수요)
```

## 9. 한계 (Limitations)

- **상관이지 인과가 아님:** 관측 데이터 기반 예측이다. 기술 발전·품종 개량·관개가 온도 효과와 섞일 수 있어, 연도 효과·카운티 고정효과로 완화하되 "실험적 인과"로 과대 해석하지 않는다.
- **시나리오 외삽:** 기후 예측은 학습 데이터 밖으로의 외삽이다. 트리 모델은 외삽에 약하므로 이 한계를 명시하고, 단일 예측이 아니라 시나리오 범위로 다룬다.
- **현재 배분 근사:** 작물별 실제 재배 면적이 없어 x_bar를 근사한다(재배 여부 + 농지 규모). 결과 해석 시 가정을 밝힌다.
- **메타휴리스틱 공정 비교:** GA/SA/TS 비교 시 파라미터 튜닝의 공정성이 결론을 좌우한다. 각 방법을 동일 조건·다회 반복으로 평가한다.
- **결측·지역 편중:** 작물별로 지역을 좁혀야 하므로 결론의 일반화 범위가 제한된다.
- **단위·정의 확인:** 수확량 단위와 온도 노출 값의 정확한 정의는 최종 해석 전 원출처로 확인한다.

## 11. 예상 질의응답 (발표·검토 대비)

**Q1. 미국은 식량 안보 측면에서 세계에서 가장 안전하지 않은가? 부족보다는 과잉·저장·가격 문제에 더 가깝지 않은가?**

두 가지로 답한다. 첫째, 미국은 세계 곡물 시장의 최대 공급자라 미국 생산량이 흔들리면 수입에 의존하는 다른 나라들의 식량 가격·안보가 출렁인다. 미국 수확량은 미국만의 문제가 아니라 글로벌 공급망의 문제다. 둘째, 이 프로젝트가 다루는 것은 '부족을 막자'가 아니라 '기온이 오르는 온난화 상황에서 어디에 뭘 심는 게 최적인가'라는 효율·적응의 문제다. 과잉이든 아니든 같은 땅에서 기후 변화에 맞춰 생산을 재배치하는 문제는 여전히 의미 있고, 최적화 문제로 딱 맞는다.

**Q2. 이 최적화가 사람을 이주시키거나 농경지를 갈아엎게 만드는 거라면, 전환 비용(장비·인프라) 때문에 오히려 손해 아닌가?**

당장 다 바꾸자는 게 아니라 '방향'에 대한 지침이다. 농가는 어차피 매년 뭘 심을지 다시 정하고(같은 밭에서 옥수수→대두 전환은 흔하다), 새로 심을 때 어느 쪽으로 조금씩 움직이는 게 유리한지를 보는 것이다. 땅을 옮기는 게 아니라 **작물 선택을 조정**하는 정도다. 더 정직하게는 **전환 비용을 최적화 문제 안에 넣는다.** 현재 배분에서 많이 벗어날수록 벌점을 주는 항을 목적함수에 추가하면 '생산은 늘리되 현재에서 너무 멀어지지 않는' 현실적인 해가 나온다. '이론적 최적'과 '전환비용 감안 최적'을 비교하는 것 자체가 좋은 분석 포인트다.

> v3 보충: 단작(單作) 설계에서는 이 "작물 선택 조정"이 문제의 본질 그 자체가 된다. 각 카운티가 주력 작물 하나를 고르므로, 최적화 결과가 곧 "이 카운티는 옥수수 대신 대두로"라는 작물 전환 지침으로 읽힌다.

**Q3. 전환 비용 항은 목적함수에 어떤 형태로 추가되나?**

원래 목적함수가 '총생산 최대화'였다면, '현재 각 카운티가 무엇을 얼마나 심고 있는지'에서 벗어난 정도에 벌점을 주는 항을 뺀다. 쉽게 말해 '지금과 많이 다르게 바꾸면 벌점'을 더하는 것이다. 대략 `[총생산 − lambda × 지금이랑 바뀐 양]` 형태다. 그러면 솔버가 '수확도 늘리고 싶지만 너무 많이 바꾸면 벌점이니 적당히 바꾸자'라고 알아서 조절한다.

> v3 보충: 연속 배분(§6-3)에서 "바뀐 양"은 `|면적 차이|`이고, 단작(§6-2)에서는 "**현재 작물과 다르게 고른 카운티 수**"가 된다. 개념("지금이랑 많이 바꾸면 벌점")은 동일하고, 세는 단위만 면적에서 카운티 개수로 바뀐다. 메타휴리스틱 적합도 함수에 자연스럽게 들어간다.

**Q4. 그럼 전환 비용을 위해 데이터가 더 필요하지 않나? 비용을 어떻게 계산하나?**

전환 비용이라 부르지만 계산하는 것은 돈이 아니라 '얼마나 바꿨는지', '지금과 얼마나 달라졌는지'라는 양이다. 그리고 '지금 각 밭에 뭘 심고 있는지'는 이미 `yielddata.csv`에 들어 있다(값이 있으면 재배, NA면 미재배). 추가 데이터 없이 λ 스윕만으로 "얼마나 바꾸면 얼마나 이득"을 보여줄 수 있다.

**Q5. 머신러닝 방향성은 무엇인가?**

**두 개의 모델**을 만든다. (1) **수확량 예측 모델** — 카운티의 온도 노출·강수·토양을 입력으로 작물 수확량을 맞히는 회귀 모델. 정확도만 보는 게 아니라 학습된 온도-수확량 관계가 약 29°C에서 꺾이는지도 확인하고, 최적화에 넣을 예측 수확량(y_hat)을 만든다. (2) **기후 예측 모델** — 지역·연도 추세로 미래의 온도 노출 분포를 예측해 온난화 시나리오를 생성한다. 두 번째 모델이 만든 미래 기후를 첫 번째 모델에 넣어 미래 수확량을 뽑고, 그것을 최적화에 연결한다.

> 교수님 피드백으로 v2의 "수확량 예측 하나"에서 "수확량 + 기후 예측 두 개"로 확장된 부분이다.

**Q6. 그럼 최적화 목표는 작물 생산량 최대화가 맞나?**

기본적으로는 맞다. 다만 (1) 카운티별 농지 한도 안에서, (2) 지금 심는 것과 너무 달라지지 않게(전환비용 벌점) 라는 조건이 붙는다. 즉 **제약과 전환비용을 감안한 생산량 최대화**다.

**Q7. 왜 순수 LP가 아니라 메타휴리스틱(GA/SA)까지 쓰나?**

각 카운티가 작물 하나를 고르는 단작 제약을 넣으면 배분이 연속 문제가 아니라 **조합 최적화(binary 배정)** 가 된다. 카운티 수가 많아지면 조합이 폭발해 정확한 MILP 솔버가 느려진다. 이 지점이 강의에서 배운 "MILP의 한계 → 메타휴리스틱의 필요성" 서사와 정확히 맞물린다. GA·SA로 '충분히 좋은' 작물 지도를 빠르게 찾고, 정확한 해와 근사 해의 품질·시간을 비교하는 것이 분석의 핵심 볼거리가 된다.
